# Explain Symbol

## Goal
准确解释一个符号在系统中的真实作用，帮助建立项目内的概念词典。

## When to use
- 用户问某个类、结构体、函数、宏、枚举、全局对象是干什么的
- 用户看到一个 manager / service / context / cache / handle，不知道它在系统中的位置
- 用户想区分两个名字相近的概念
- 用户需要快速理解一个 symbol 的上下游关系

## Instructions
1. 先确认 symbol 的种类：
   - class / struct
   - function / method
   - macro
   - enum / flag
   - global / singleton
   - typedef / alias
   - interface / abstract base
2. 先给一句本质定义：它在系统里负责什么。
3. 说明它处于哪一层：
   - 接口层
   - 调度层
   - 核心逻辑层
   - 数据层
   - 平台层
4. 找它的上下游关系：
   - 谁创建它
   - 谁持有它
   - 谁调用它
   - 它依赖谁
   - 它影响谁
5. 分析生命周期：
   - 何时创建
   - 何时有效
   - 何时失效
   - 何时释放
6. 分析线程归属：
   - 主线程
   - worker
   - render thread
   - 多线程共享
7. 如果它是状态/缓存/句柄，说明：
   - 它代表什么状态
   - 它的 key 是什么
   - 它何时刷新/失效
8. 如果它是函数，说明：
   - 输入
   - 输出
   - 副作用
   - 隐式前置条件
9. 如果它是宏，说明：
   - 展开后影响什么
   - 是否涉及平台分支 / 配置开关 / 编译条件
10. 区分“表面职责”和“真实职责”：
   - 名字看起来像什么
   - 实际上在项目里做什么
11. 明确列出不确定项，不要根据命名脑补。
