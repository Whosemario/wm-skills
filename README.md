# wm-skills

Skills for AI agents.

## Skills

### `system-overview`
快速理解复杂仓库或子系统，建立全局心智模型。

适用场景：
- 接手一个新模块，想先看清整体结构
- 改代码前，需要先摸清入口、分层和关键对象
- 想知道系统的主流程、线程模型和关键依赖

核心能力：
- 识别仓库或子系统边界
- 提炼启动点、初始化点、注册点和主循环
- 梳理模块分层、核心对象、线程模型与异步链
- 给出风险点和新人阅读顺序

### `trace-feature`
沿着真实调用链追踪一个功能从入口到可观察结果的执行路径。

适用场景：
- 想知道某个功能是怎么跑通的
- 已知一个函数或类，需要继续追上下游
- 改动前做影响面分析，确认会改到哪里

核心能力：
- 找到功能入口，包括事件、回调、消息或脚本入口
- 跳过低价值转发层，只保留关键调用链
- 追到具体副作用，如状态更新、资源提交、网络发送或文件读写
- 标出线程切换、异步边界、前置条件和高风险点

### `explain-symbol`
准确解释一个 symbol 在系统中的真实职责，而不是只看名字猜用途。

适用场景：
- 不清楚某个类、函数、宏、枚举、全局对象是干什么的
- 想区分两个名字相近但职责不同的概念
- 需要快速理解一个 symbol 的上下游和生命周期

核心能力：
- 判断 symbol 类型，如 class、function、macro、enum、global
- 说明它所处分层、所属模块和最接近的核心对象
- 梳理创建者、持有者、调用者、依赖项和影响范围
- 分析生命周期、线程归属、输入输出、副作用和常见误解

### `capture-web-article`
把单篇网页文章、博客或社交平台帖子归档为本地 Markdown 包，并下载正文内图片。

适用场景：
- 需要把一个网页内容保存到本地长期留档
- 需要保留渲染后的正文内容，而不是抓原始 HTML
- 页面需要登录后才能看到正文，仍然希望完整保存

核心能力：
- 要求用户提供 URL 和输出目录，并在本地创建带时间戳的归档目录
- 通过浏览器渲染后的 DOM 提取正文，而不是直接抓取原始页面
- 过滤导航、侧边栏、评论区等噪音内容，按阅读顺序保留图片
- 生成 Markdown 正文、下载图片并改写为本地引用

注意事项：
- 这个 skill 只在用户显式调用 `$capture-web-article` 时使用
- 执行前必须提供 `url` 和 `output_dir`
